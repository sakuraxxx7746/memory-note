# LoginPage UI仕様

## 共通事項

UIは特別な指定がない限り、components/uiにある、Shadcn/UIを使用すること
ボタンの要素について説明があるとき、Shadcnで実装すること。
Shadcnで実装できないものがあれば自動実装をせず、実装ができなかった旨をコメントに記載すること

## 画面概要

ユーザーがメールアドレスとパスワードを入力し、
ログインを行うための画面。

Next.js App Router を使用し、
ページコンポーネントとして実装する。

---

## 画面構成

LoginPage は以下の要素で構成される。

- Title
- EmailInput
- PasswordInput
- LoginButton

上から縦方向に並ぶ。

---

## 各要素の仕様

### 1. Title

- 種別: テキスト
- 表示内容: 「ログイン」
- 画面の最上部に配置される
- 役割: 画面の目的をユーザーに示す

---

### 2. EmailInput

- 種別: 入力欄
- input type: email
- placeholder: 「メールアドレス」
- 必須入力
- 値はログイン処理に使用される

---

### 3. PasswordInput

- 種別: 入力欄
- input type: password
- placeholder: 「パスワード」
- 必須入力
- セキュリティのためマスク表示する

---

### 4. LoginButton

- 種別: ボタン
- 表示内容: 「ログイン」
- クリック時にログイン処理を実行する
- EmailInput / PasswordInput が未入力の場合は無効化してもよい
- 認証の仕様はguidlines/auth-spec.mdファイルを参照すること
- 認証に成功した場合、アラートで認証の返り値を出すこと
- 戦勝に失敗した場合、アラートでエラーを表示すること

---

## 振る舞い（最低限）

- ボタン押下時に入力値を取得する
- 実際の認証処理はこの画面では実装しない
- onClick ハンドラは用意する

---

## 実装方針

- Next.js App Router を使用
- page.tsx として実装
- スタイリングは最低限でよい
- 見た目より構造と責務を優先する
